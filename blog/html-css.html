Using Classes vs. IDs

Using Classes vs. IDs

Classes are referenced in CSS by putting a dot in front of property names. Example: .my-class{ }

IDs are referenced in CSS by putting a hash sign in front of a property name. Example: #my-id{ }

Usage

It is best practice to only use IDs if you want a style to apply to one element on the web page, and to use classes if you want a style to apply to multiple elements.

  • IDs are not reusable, whereas classes are reusable.
  • Each HTML <element> can have only one ID, but can have multiple classes.
  • Each HTML page can have only one <element> with a specific ID, whereas a specific class can be used in multiple elements.

Using JavaScript

The "getElementById" function is commonly used in JavaScript.

By assigning an ID to a specific element this allows JavaScript to interact with the element in question through the use of this function.

Using IDs for Internal Links

If you want to take users to a location on the current page you can do so by setting the href property in an anchor tag to an element's ID. When clicked the browser will automatically scroll the page to show that element.

A Note on Specificity

Specificity refers to the means by which browsers decide which CSS property values are the most relevant to an element, and which should be applied.

IDs have higher specificity than classes, so if both are applied to the same element and have conflicting styles, the styles of the id will be applied.
